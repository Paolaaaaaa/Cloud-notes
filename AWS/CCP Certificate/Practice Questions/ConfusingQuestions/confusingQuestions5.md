## Question 9:
Denial-of-Service (DoS) attack comes from different sources.

- A) False     
- B) True

Response: A
> False: Because a Denial-of-Service (DoS) attack typically originates from a single source, whereas a Distributed Denial-of-Service (DDoS) attack comes from multiple sources.

## Question 13:
Based on application's network requests, AWS Web Application Firewall can block network traffic.

- A) False     
- B) True

Response: B
> True: AWS Web Application Firewall (WAF) can indeed block network traffic based on the application's network requests. It allows you to create custom rules to filter and monitor HTTP and HTTPS requests, helping to protect your web applications from common web exploits and attacks.

## Question 14:
AWS Elastic Block Store Snapshot is:

-  A) Incremental data backup     
- B) Differential data backup
- C) Full data backup

Response: A
> Incremental data backup: AWS Elastic Block Store (EBS) Snapshots are incremental backups, meaning that only the blocks of data that have changed since the last snapshot are saved. This approach helps to reduce storage costs and speeds up the backup process.

## Question 16:
What is AWS DynamoDB Accelerator capable of?

- A) Improves read times of non-relational data     
- B) Improves visualisation times of graphs
- C) Improves write times of documents
- D) Improves read times of relational data

Response: A

DAX can reduce read latency from milliseconds to microseconds, making it ideal for applications that require fast response times.

## Question 17:

- B) AWS Neptune
- C) AWS ElastiCache
- D) AWS DocumentDB

Response: A

## Question 20:
If an instance store reboots, does the data in the instance persist?

- A) No
- B) Yes

Response: A
> No: Instance store volumes are ephemeral storage that is physically attached to the host computer. If the instance is stopped, terminated, or reboots, the data stored in the instance store is lost. For persistent storage, it is recommended to use Amazon EBS (Elastic Block Store) volumes.
