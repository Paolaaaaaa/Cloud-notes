# Security in VPCs

## Security Groups
***Definition: Security groups act as virtual firewalls for your resources to control inbound and outbound traffic.***

Define rules that allow traffic to and from your instances based on protocols, ports and sources/ IP destinations.
> Statefull: If an incoming request is allowed, the response is automatically allowed, regardless of outbound rules.
-  Automatic response to allowed inbound traffic.
## NACLs
***Definition: Network Access Control Lists (NACLs) are an additional layer of security that***
> stateless: Responses to allowed inbound traffic are subject to the rules for outbound traffic (and vice versa).
- Should explicitly allow both inbound and outbound traffic.
