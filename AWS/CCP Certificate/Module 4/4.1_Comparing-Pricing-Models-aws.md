## 4.1 Comparing Pricing Models in AWS  
### Cost Optimization

Cost optimization is the ability to **run systems that deliver business value at the lowest possible cost**.

---

## Amazon S3 Pricing Models (Storage Classes)

### Amazon S3 Standard
- General-purpose object storage
- Designed for **frequently accessed data**
- High durability and availability
- Higher storage cost, lower access cost

**Use cases:**
- Active data
- Web content
- Data lakes
- Mobile and web applications

---

### Amazon S3 Glacier Instant Retrieval
- Low-cost storage for infrequently accessed data
- **Millisecond retrieval**
- More expensive than Deep Archive, cheaper than Standard

**Use cases:**
- Backup data accessed occasionally

---

### Amazon S3 Glacier Flexible Retrieval
- Low-cost archival storage
- Retrieval time: minutes to hours

**Use cases:**
- Archives that do not need immediate access

---

### Amazon S3 Glacier Deep Archive
- **Lowest-cost storage option**
- **Slow retrieval** (hours)
- Designed for long-term data retention

**Use cases:**
- Compliance data
- Long-term backups
- Archival data rarely accessed

---

## Data Transfer Costs


| Data Transfer Type | Cost |
|------------------|------|
| Inbound data (Internet → AWS) | Usually free |
| Outbound data (AWS → Internet) | Charged |
| Data transfer within same AZ | Free |
| Data transfer between AZs | Charged |
